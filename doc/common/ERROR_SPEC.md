# CampusCrew 공통 에러코드 명세서

## 1. 문서 목적

이 문서는 CampusCrew의 모든 API에서 사용하는 에러 응답의 HTTP 상태 코드, 에러 메시지, 발생 조건을 고정한다.  
프론트엔드, 백엔드, AI 서버 모두 이 문서를 기준으로 에러 처리를 통일한다.

## 2. 공통 에러 응답 형식

모든 에러 응답은 아래 구조를 따른다.

```json
{
  "success": false,
  "data": null,
  "message": "에러 메시지"
}
```

- `success`는 항상 `false`다.
- `data`는 항상 `null`이다.
- `message`는 사용자에게 보여줄 수 있는 한국어 문장이다.
- 에러 응답 구조는 도메인에 관계없이 동일하다.

## 3. HTTP 상태 코드 기준

| 상태 코드 | 의미                  | 사용 기준                                             |
| --------- | --------------------- | ----------------------------------------------------- |
| `400`     | Bad Request           | 요청 본문의 형식, 필수값, 제약 조건이 잘못된 경우     |
| `401`     | Unauthorized          | 인증 정보가 없거나 유효하지 않은 경우                 |
| `403`     | Forbidden             | 인증은 되었지만 해당 리소스에 접근할 권한이 없는 경우 |
| `404`     | Not Found             | 요청한 리소스가 존재하지 않는 경우                    |
| `409`     | Conflict              | 이미 존재하는 데이터와 충돌하는 경우                  |
| `500`     | Internal Server Error | 서버 내부 오류가 발생한 경우                          |

## 4. 에러코드 상세

### 4.1 인증 에러 (401)

| 코드       | 메시지                           | 발생 조건                                  | 관련 API               |
| ---------- | -------------------------------- | ------------------------------------------ | ---------------------- |
| `AUTH_001` | 로그인 정보가 올바르지 않습니다. | 이메일 또는 비밀번호가 틀린 경우           | `POST /api/auth/login` |
| `AUTH_002` | 인증 토큰이 없습니다.            | Authorization 헤더가 누락된 경우           | 모든 보호 API          |
| `AUTH_003` | 인증 토큰이 만료되었습니다.      | JWT 토큰이 만료된 경우                     | 모든 보호 API          |
| `AUTH_004` | 유효하지 않은 인증 토큰입니다.   | JWT 토큰이 변조되었거나 형식이 잘못된 경우 | 모든 보호 API          |

### 4.2 권한 에러 (403)

| 코드            | 메시지                            | 발생 조건                                            | 관련 API                         |
| --------------- | --------------------------------- | ---------------------------------------------------- | -------------------------------- |
| `FORBIDDEN_001` | 해당 팀에 접근할 권한이 없습니다. | 팀 멤버가 아닌 사용자가 팀 하위 리소스에 접근한 경우 | 모든 `/api/teams/{teamId}/*` API |
| `FORBIDDEN_002` | 작성자만 수정할 수 있습니다.      | 작성자가 아닌 사용자가 리소스를 수정하려는 경우      | 공지/일정/할 일/회의록 수정 API  |
| `FORBIDDEN_003` | 작성자만 삭제할 수 있습니다.      | 작성자가 아닌 사용자가 리소스를 삭제하려는 경우      | 공지/일정/할 일/회의록 삭제 API  |

### 4.3 리소스 없음 에러 (404)

| 코드            | 메시지                      | 발생 조건                    | 관련 API                           |
| --------------- | --------------------------- | ---------------------------- | ---------------------------------- |
| `NOT_FOUND_001` | 존재하지 않는 사용자입니다. | 요청한 사용자 ID가 없는 경우 | 할 일 담당자 지정 등               |
| `NOT_FOUND_002` | 존재하지 않는 팀입니다.     | 요청한 팀 ID가 없는 경우     | 모든 `/api/teams/{teamId}/*` API   |
| `NOT_FOUND_003` | 존재하지 않는 공지입니다.   | 요청한 공지 ID가 없는 경우   | 공지 상세/수정/삭제 API            |
| `NOT_FOUND_004` | 존재하지 않는 일정입니다.   | 요청한 일정 ID가 없는 경우   | 일정 상세/수정/삭제 API            |
| `NOT_FOUND_005` | 존재하지 않는 할 일입니다.  | 요청한 할 일 ID가 없는 경우  | 할 일 상세/수정/삭제/상태 변경 API |
| `NOT_FOUND_006` | 존재하지 않는 회의록입니다. | 요청한 회의록 ID가 없는 경우 | 회의록 상세/수정/삭제 API          |

### 4.4 검증 에러 (400)

## 7. 프론트엔드 에러 처리 기준

| 상태 코드 | 프론트엔드 처리                                                          |
| --------- | ------------------------------------------------------------------------ |
| `400`     | 입력 폼 근처에 에러 메시지를 표시한다.                                   |
| `401`     | 저장된 토큰을 제거하고 로그인 화면으로 이동한다.                         |
| `403`     | 팀 목록 화면으로 이동하거나 권한 없음 안내를 표시한다.                   |
| `404`     | 목록 화면으로 이동하거나 존재하지 않음 안내를 표시한다.                  |
| `409`     | 입력 폼 근처에 충돌 메시지를 표시한다.                                   |
| `500`     | "서버 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." 안내를 표시한다. |
